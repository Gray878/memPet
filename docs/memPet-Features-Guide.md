## 产品概述

memPet 是一款智能桌面宠物应用，具备持续记忆能力和主动推理功能。它不仅能陪伴用户聊天，还能主动关心用户的工作和生活状态，提供个性化的建议和提醒。

### 核心价值

- 持续记忆：记住所有对话和用户行为，建立长期记忆
- 主动关怀：根据用户状态主动提供建议和提醒
- 个性化：4种预设性格 + 自定义配置
- 上下文感知：智能检测用户工作状态和环境
- 本地优先：数据存储在本地，保护隐私

## 核心功能

### 1. 智能对话系统

功能描述：
- 支持自然语言对话
- 基于历史记忆的上下文理解
- 根据用户状态调整回复风格

使用场景示例：
```
用户："我今天做了什么？"
memPet："你今天很充实呢！上午专注写代码，下午参加了会议，晚上还在学习文档"
```

技术实现：
- 调用 LLM API (Claude/OpenAI)
- 从 memU-server 检索相关记忆
- 结合当前上下文生成回复

---

### 2. 持续记忆系统

#### 2.1 自动记忆

功能描述：
- 自动记录用户对话内容
- 监控用户应用使用情况
- 追踪工作时长和休息次数

记忆类型：
- 对话记忆：用户与 memPet 的所有对话
- 行为记忆：应用切换、工作时长等
- 观察记忆：系统自动记录的用户活动

数据存储：
```
存储位置：%APPDATA%/memPet/memU/
- conversation-*.json  # 对话记录
- memory.db           # SQLite 数据库
```

#### 2.2 记忆检索

功能描述：
- 全文搜索历史记忆
- 按时间范围筛选
- 智能关联相关记忆

使用场景：
```
用户："上周我说过要学什么？"
memPet 检索记忆 → 找到相关对话 → 回复："你说要学 Rust 编程"
```

### 3. 性格系统

#### 3.1 预设性格

友好型 (Friendly)
- 特点：温和友善，总是关心用户
- 语气：温暖亲切，使用"呀"、"啦"等语气词
- 示例："主人辛苦啦~ 看到你一直在写代码，记得休息眼睛哦"

活力型 (Energetic)
- 特点：充满活力和热情
- 语气：大量感叹号，丰富的表情
- 示例："冲鸭主人！今天也要元气满满！"

专业型 (Professional)
- 特点：专业简洁，重视效率
- 语气：正式但友好，基于数据
- 示例："检测到连续工作2小时。建议：休息10分钟，活动一下。"

傲娇型 (Tsundere)
- 特点：表面高冷，内心温柔
- 语气：先否定再转折
- 示例："哼，又来找我了？...算了，看在你那么努力的份上，休息一下吧。"

#### 3.2 性格配置

可调节参数：
- Emoji 使用频率：0-100%
- 主动性级别：0-100%
- 正式程度：0-100%
- 自定义特质：添加个性化标签

### 4. 上下文感知系统

#### 4.1 时间维度

检测内容：
- 当前时间和日期
- 是否工作时间 (9:00-18:00)
- 是否深夜 (22:00-6:00)
- 星期几

应用场景：
- 深夜工作 → 提醒休息
- 周末加班 → 关心问候

#### 4.2 工作状态

检测内容：
- 持续工作时长
- 空闲时长
- 今日休息次数
- 当前活动应用

状态分类：
- Working：工作中
- Studying：学习中
- Relaxing：休闲中
- Gaming：游戏中
- Idle：空闲

#### 4.3 专注度检测

专注级别：
- DeepFocus：深度专注（勿扰模式/全屏）
- NormalFocus：正常专注
- Distracted：分心（频繁切换）
- Idle：空闲

应用场景：
- 深度专注 → 保持安静，不打扰
- 分心状态 → 提供专注建议

#### 4.4 疲劳度检测

疲劳级别：
- Fresh：精力充沛 (0-1小时)
- Normal：正常 (1-2小时)
- Tired：疲劳 (2-4小时)
- Exhausted：极度疲劳 (4小时+)

应用场景：
- 疲劳 → 建议休息
- 极度疲劳 → 主动提醒并关心

#### 4.5 会议检测

检测方式：
- 识别会议软件 (Zoom, Teams, 腾讯会议等)
- 检测摄像头状态
- 检测屏幕共享状态

应用场景：
- 会议中 → 静默模式，不打扰

### 5. 行为系统

#### 5.1 基础行为

- Idle：待机状态，呼吸动画
- Walking：在屏幕上行走
- Sleeping：睡觉状态
- Waving：挥手打招呼

#### 5.2 互动行为

- Thinking：思考中（处理用户请求时）
- Talking：说话（回复用户时）
- Celebrating：庆祝（用户完成任务）
- Worrying：担心（用户疲劳时）
- Comforting：安慰（用户情绪低落）

#### 5.3 主动行为

- Reminding：提醒（休息、喝水等）
- Suggesting：建议（工作方法、效率提升）
- Observing：观察（静默监控用户状态）

#### 5.4 趣味行为

- Dancing：跳舞
- Playing：玩耍
- Stretching：伸懒腰
- Yawning：打哈欠

#### 5.5 行为决策逻辑

```
深度专注模式 → Sleeping (保持安静)
会议中 → Observing (静默观察)
极度疲劳 → Worrying (主动关心)
深夜 → Yawning (困倦状态)
用户空闲 → Walking/Waving (互动)
```

### 6. 主动推理系统

#### 6.1 监控循环

运行频率：每 5 分钟检查一次

检测内容：
1. 收集最近记忆
2. 分析用户行为模式
3. 检测异常状态
4. 生成主动建议

#### 6.2 干预场景

疲劳提醒：
```
条件：连续工作 > 2 小时
行为：Worrying → Reminding
内容："主人已经专注编程2小时了，要不要休息一下眼睛呀？"
```

深夜工作：
```
条件：22:00 后仍在工作
行为：Worrying
内容："已经很晚了，早点休息吧，明天继续加油！"
```

长时间空闲：
```
条件：空闲 > 30 分钟
行为：Waving
内容："主人，要不要聊聊天？"
```

#### 6.3 模式学习

学习内容：
- 用户工作习惯
- 休息偏好
- 对话风格
- 兴趣爱好

应用场景：
```
发现：用户每天 21:00 复盘一天工作
学习：设置定时提醒
执行：每天 21:00 主动询问"今天过得怎么样？"
```

## 功能流程图

### 1. 用户对话流程

```
用户输入消息
    |
    v
[渲染进程] ChatPanel
    |
    | 1. 显示用户消息
    | 2. 显示"思考中..."
    | 3. 调用 IPC
    |
    v
[主进程] IPC Handler
    |
    | 1. 接收消息
    | 2. 获取用户上下文
    |
    v
[主进程] MemUService
    |
    | HTTP POST /retrieve
    | 检索相关记忆
    |
    v
[memU-server]
    |
    | 查询数据库
    | 返回记忆列表
    |
    v
[主进程] 构建完整上下文
    |
    | - 用户消息
    | - 相关记忆
    | - 当前状态
    | - 性格配置
    |
    v
[主进程] LLM Client
    |
    | 调用 Claude/OpenAI API
    | 生成回复
    |
    v
[主进程] MemUService
    |
    | HTTP POST /memorize
    | 存储对话记录
    |
    v
[主进程] IPC 返回结果
    |
    v
[渲染进程] ChatPanel
    |
    | 1. 显示 AI 回复
    | 2. 触发说话动画
    |
    v
完成
```

### 2. 系统监控流程

```
[主进程] SystemMonitor
    |
    | 每 30 秒执行一次
    |
    v
获取系统状态
    |
    +- 活动窗口信息
    +- 空闲时间
    +- 屏幕状态
    +- 系统资源
    |
    v
检测上下文变化
    |
    | 窗口切换？
    | 工作时长？
    | 状态改变？
    |
    v
[是] 记录活动
    |
    | HTTP POST /memorize
    | 存储观察记录
    |
    v
[memU-server] 保存记忆
    |
    v
继续监控循环
```

### 3. 主动推理流程

```
[主进程] Memory Agent
    |
    | 每 5 分钟执行一次
    |
    v
收集最近记忆
    |
    | HTTP POST /retrieve
    | 查询最近 30 分钟活动
    |
    v
模式检测
    |
    | 分析用户行为
    | 识别异常状态
    |
    v
[检测到问题]
    |
    | 例如：连续工作 2 小时
```

## 技术特性

### 1. 双 Agent 架构

UI Agent (主执行引擎)
- 职责：处理用户交互，执行任务
- 特点：响应式，快速反应
- 运行：主线程

Memory Agent (记忆智能引擎)
- 职责：监控、记忆、推理
- 特点：主动式，后台运行
- 运行：独立线程

### 2. 事件驱动通信

事件类型：
- UserMessage：用户消息
- UserAction：用户操作
- MemoryStored：记忆已存储
- PatternDetected：模式检测
- ProactiveSuggestion：主动建议
- PersonalityChanged：性格变更

### 3. 本地化存储

数据目录：
```
Windows: C:\Users\{用户}\AppData\Roaming\memPet\
+-- memU\
|   +-- conversation-*.json
|   +-- memory.db
+-- config\
|   +-- settings.json
|   +-- personality.json
+-- logs\
    +-- app.log
```

### 4. 性能优化

批量处理：
- 记忆写入批处理（10条/批）
- 定时刷新（60秒）

缓存机制：
- LRU 缓存常用查询
- TTL 5 分钟

节流控制：
- 主动推理最小间隔 5 分钟
- 避免频繁 LLM 调用

### 5. 容错机制

服务降级：
- memU-server 连接失败 → 本地队列缓存
- LLM API 超时 → 使用缓存模板
- Memory Agent 崩溃 → UI Agent 继续工作

错误恢复：
- 自动重连
- 离线队列
- 数据备份
