**响应：**

    "message": "Hello MemU user!",

    "status": "running"

## 记忆管理接口

### POST /memorize

存储记忆（支持对话和系统观察两种类型）

#### 对话记忆

**请求：**
```json
{
    "type": "conversation",
    "content": [
        {
            "role": "user",
            "content": {"text": "我今天工作了 8 小时"},
            "created_at": "2024-02-15T10:00:00"
        },
        {
            "role": "assistant",
            "content": {"text": "辛苦了，记得休息"},
            "created_at": "2024-02-15T10:00:05"
        }
    ]
}
```

**字段说明：**
- `type`：固定为 `"conversation"`
- `content`：对话内容数组
  - `role`：角色，可选值：`user`、`assistant`、`system`
  - `content.text`：消息文本
  - `created_at`：创建时间（ISO 8601 格式）

#### 系统观察记录

**请求：**
```json
{
    "type": "system_observation",
    "observation": {
        "type": "app_switch",
        "app": "VSCode",
        "duration": 3600,
        "timestamp": "2024-02-15T10:00:00"
    }
}
```

**字段说明：**
- `type`：固定为 `"system_observation"`
- `observation`：观察记录对象
  - `type`：观察类型（见下表）
  - `timestamp`：时间戳（ISO 8601 格式）
  - 其他字段根据观察类型而定

**支持的观察类型：**

| 类型 | 说明 | 必需字段 | 格式化结果示例 |
|------|------|----------|----------------|
| `app_switch` | 应用切换 | `app`, `duration` | "用户在 VSCode 工作了 60 分钟" |
| `fatigue_detected` | 疲劳检测 | `hours`, `level` | "检测到用户疲劳：连续工作 2 小时" |
| `idle_detected` | 空闲检测 | `minutes` | "用户空闲了 30 分钟" |
| `work_session_end` | 工作会话结束 | `app`, `duration` | "用户结束了在 VSCode 的工作，持续 60 分钟" |
| `late_night_work` | 深夜工作 | `hour` | "检测到深夜工作：23:00 仍在活动" |
| `break_taken` | 休息记录 | `duration` | "用户休息了 10 分钟" |

**响应：**
```json
{
    "status": "success",
    "result": {
        "conversation_id": "uuid",
        "items_created": 3,
        "categories_updated": 2
    }
}
```

**状态码：**
- 200：成功
- 400：请求参数错误
- 500：服务器内部错误

### POST /retrieve

检索记忆（支持对话和主动推理两种场景）

#### 对话场景

用于用户主动提问时检索相关记忆。

**请求：**
```json
{
    "scenario": "conversation",
    "query": "用户今天的工作情况",
    "limit": 3
}
```

**字段说明：**
- `scenario`：固定为 `"conversation"`
- `query`：查询文本
- `limit`：返回记忆数量限制（可选，默认 5）

#### 主动推理场景

用于主动推理时根据上下文检索相关记忆。

**请求：**
```json
{
    "scenario": "proactive",
    "context": {
        "working_duration": 7200,
        "active_app": "VSCode",
        "fatigue_level": "Tired",
        "is_late_night": false,
        "idle_time": 0,
        "is_work_hours": true,
        "focus_level": "NormalFocus"
    },
    "limit": 5
}
```

**字段说明：**
- `scenario`：固定为 `"proactive"`
- `context`：系统上下文对象（见[数据模型](#systemcontext)）
- `limit`：返回记忆数量限制（可选，默认 5）

**响应：**
```json
{
    "status": "success",
    "result": {
        "items": [
            {
                "id": "uuid",
                "content": "用户在 VSCode 工作了 60 分钟",
                "summary": "工作记录",
                "category": "工作习惯",
                "timestamp": 1708000000
            }
        ]
    }
}
```

**状态码：**
- 200：成功
- 400：请求参数错误
- 500：服务器内部错误

## 主动推理接口

### POST /proactive/analyze

分析当前上下文，生成建议列表

**请求：**
```json
{
    "context": {
        "working_duration": 7200,
        "active_app": "VSCode",
        "fatigue_level": "Tired",
        "is_late_night": false,
        "idle_time": 0,
        "is_work_hours": true,
        "focus_level": "NormalFocus"
    }
}
```

**字段说明：**
- `context`：系统上下文对象（见[数据模型](#systemcontext)）

**响应：**
```json
{
    "status": "success",
    "suggestions": [
        {
            "type": "fatigue_reminder",
            "priority": "high",
            "message": "检测到疲劳",
            "reason": "连续工作 2 小时",
            "action": "建议休息 10 分钟"
        },
        {
            "type": "break_suggestion",
            "priority": "medium",
            "message": "定时休息提醒",
            "reason": "已工作 2 小时",
            "action": "建议站起来活动一下"
        }
    ]
}
```

**建议类型：**
- `fatigue_reminder`：疲劳提醒（冷却时间：30 分钟）
- `late_night_reminder`：深夜提醒（冷却时间：1 小时）
- `idle_interaction`：空闲互动（冷却时间：30 分钟）
- `break_suggestion`：休息建议（冷却时间：30 分钟）
- `hydration_reminder`：喝水提醒（冷却时间：1 小时）

**优先级：**
- `high`：高优先级（需要立即处理）
- `medium`：中优先级（建议处理）
- `low`：低优先级（可选处理）

**状态码：**
- 200：成功
- 500：服务器内部错误

### POST /proactive/generate

根据建议和相关记忆，生成自然语言消息

**请求：**
```json
{
    "suggestion": {
        "type": "fatigue_reminder",
        "priority": "high",
        "message": "检测到疲劳",
        "reason": "连续工作 2 小时",
        "action": "建议休息 10 分钟"
    },
    "context": {
        "working_duration": 7200,
        "fatigue_level": "Tired"
    },
    "personality": "friendly",
    "limit": 3
}
```

**字段说明：**
- `suggestion`：建议对象（来自 `/proactive/analyze`）
- `context`：系统上下文对象
- `personality`：性格类型（见下表）
- `limit`：检索记忆数量限制（可选，默认 3）

**性格类型：**

| 类型 | 说明 | 语气特点 | 示例 |
|------|------|----------|------|
| `friendly` | 友好型 | 温和友善，使用"呀"、"啦"等语气词 | "主人辛苦啦~ 记得休息眼睛哦" |
| `energetic` | 活力型 | 充满活力，大量感叹号和表情 | "冲鸭主人！今天也要元气满满！" |
| `professional` | 专业型 | 专业简洁，基于数据 | "检测到连续工作2小时。建议：休息10分钟" |
| `tsundere` | 傲娇型 | 表面高冷，内心温柔 | "哼，又来找我了？...算了，休息一下吧" |

**响应：**
```json
{
    "status": "success",
    "message": "主人已经工作 2 小时了，要不要休息一下眼睛呀？",
    "memories": [
        {
            "id": "uuid",
            "content": "用户通常每 2 小时休息一次",
            "category": "工作习惯"
        }
    ],
    "prompt": "..."
}
```

**字段说明：**
- `message`：生成的自然语言消息
- `memories`：相关记忆列表
- `prompt`：使用的 LLM prompt（调试用）

**状态码：**
- 200：成功
- 500：服务器内部错误

### GET /proactive/cooldown

获取所有建议类型的冷却状态

**请求：** 无参数

**响应：**
```json
{
    "status": "success",
    "cooldowns": {
        "fatigue_reminder": {
            "cooldown_seconds": 1800,
            "remaining_seconds": 600,
            "can_trigger": false,
            "last_triggered": "2024-02-15T10:00:00"
        },
        "late_night_reminder": {
            "cooldown_seconds": 3600,
            "remaining_seconds": 0,
            "can_trigger": true,
            "last_triggered": null
        }
    }
}
```

**字段说明：**
- `cooldown_seconds`：冷却时间（秒）
- `remaining_seconds`：剩余冷却时间（秒）
- `can_trigger`：是否可以触发
- `last_triggered`：上次触发时间（ISO 8601 格式）

**状态码：**
- 200：成功

---

### POST /proactive/cooldown/reset

重置冷却时间

**请求：**

    "type": "fatigue_reminder"

**字段说明：**
- `type`：建议类型（可选，不提供则重置所有）

**响应：**
```json
{
    "status": "success",
    "message": "已重置冷却时间: fatigue_reminder"
}
```

**状态码：**
- 200：成功

## 批量处理接口

### POST /batch/observations

批量添加观察记录到缓冲区

用于高频监控数据的批量处理，避免频繁调用 `/memorize`。

**请求：**
```json
{
    "observations": [
        {
            "type": "app_switch",
            "app": "VSCode",
            "duration": 300,
            "timestamp": "2024-02-15T10:00:00"
        },
        {
            "type": "idle_detected",
            "minutes": 5,
            "timestamp": "2024-02-15T10:05:00"
        }
    ]
}
```

**字段说明：**
- `observations`：观察记录数组

**响应：**
```json
{
    "status": "success",
    "message": "已添加 2 条观察记录到缓冲区",
    "buffer_size": 5
}
```

**字段说明：**
- `buffer_size`：当前缓冲区大小

**状态码：**
- 200：成功
- 500：服务器内部错误

**注意事项：**
- 缓冲区会在以下情况自动刷新：
  - 缓冲区满 10 条记录
  - 距离上次刷新超过 5 分钟
- 数据会被自动聚合和过滤

---

### POST /batch/flush

强制刷新缓冲区

立即将缓冲区中的所有观察记录存储到数据库。

**请求：** 无参数

**响应：**
```json
{
    "status": "success",
    "message": "已刷新 5 条观察记录"
}
```

**状态码：**
- 200：成功
- 500：服务器内部错误

**使用场景：**
- 应用关闭时
- 需要立即保存数据时

## 数据模型

### SystemContext

系统上下文对象，用于主动推理场景。

```typescript
interface SystemContext {
    working_duration: number;      // 工作时长（秒）
    active_app: string;            // 当前活动应用
    fatigue_level: string;         // 疲劳级别
    is_late_night: boolean;        // 是否深夜（22:00-6:00）
    idle_time: number;             // 空闲时间（秒）
    is_work_hours?: boolean;       // 是否工作时间（9:00-18:00）
    focus_level?: string;          // 专注级别
}
```

**疲劳级别（fatigue_level）：**
- `Fresh`：精力充沛（0-1 小时）
- `Normal`：正常（1-2 小时）
- `Tired`：疲劳（2-4 小时）
- `Exhausted`：极度疲劳（4 小时以上）

**专注级别（focus_level）：**
- `DeepFocus`：深度专注（勿扰模式/全屏）
- `NormalFocus`：正常专注
- `Distracted`：分心（频繁切换）
- `Idle`：空闲

---

### Suggestion

建议对象，由 `/proactive/analyze` 生成。

```typescript
interface Suggestion {
    type: string;                  // 建议类型
    priority: string;              // 优先级：high, medium, low
    message: string;               // 建议消息
    reason: string;                // 原因
    action: string;                // 建议的操作
}
```

---

### SystemObservation

系统观察记录对象。

```typescript
interface SystemObservation {
    type: string;                  // 观察类型
    timestamp: string;             // 时间戳（ISO 8601）
    [key: string]: any;            // 其他字段根据类型而定
}
```

**示例：**

```typescript
// 应用切换
{
    type: "app_switch",
    app: "VSCode",
    duration: 3600,
    timestamp: "2024-02-15T10:00:00"
}

// 疲劳检测
{
    type: "fatigue_detected",
    hours: 2,
    level: "Tired",
    timestamp: "2024-02-15T12:00:00"
}
```

## 错误处理

所有接口在出错时返回统一的错误格式：

```json
{
    "detail": "错误信息"
}
```

**HTTP 状态码：**
- 200：成功
- 400：请求参数错误
- 500：服务器内部错误

**常见错误：**

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| "不支持的记忆类型: xxx" | `type` 字段值不正确 | 使用 `conversation` 或 `system_observation` |
| "不支持的检索场景: xxx" | `scenario` 字段值不正确 | 使用 `conversation` 或 `proactive` |
| "对话场景需要提供 'query' 参数" | 缺少 `query` 字段 | 添加 `query` 字段 |
| "MemorizeMixin.memorize() takes..." | 内部错误 | 检查服务器日志 |

## 使用示例

### 完整工作流程

#### 1. 系统监控 → 存储观察

```python
import requests
from datetime import datetime

BASE_URL = "http://localhost:8000"

# 检测到应用切换
observation = {
    "type": "app_switch",
    "app": "VSCode",
    "duration": 3600,
    "timestamp": datetime.now().isoformat()
}

# 存储观察记录
response = requests.post(f"{BASE_URL}/memorize", json={
    "type": "system_observation",
    "observation": observation
})

print(response.json())
```

#### 2. 主动推理 → 生成建议

```python
# 收集系统上下文
context = {
    "working_duration": 7200,
    "active_app": "VSCode",
    "fatigue_level": "Tired",
    "is_late_night": False,
    "idle_time": 0
}

# 分析上下文
response = requests.post(f"{BASE_URL}/proactive/analyze", json={
    "context": context
})

suggestions = response.json()["suggestions"]

if suggestions:
    # 选择优先级最高的建议
    suggestion = suggestions[0]
    
    # 生成自然语言消息
    response = requests.post(f"{BASE_URL}/proactive/generate", json={
        "suggestion": suggestion,
        "context": context,
        "personality": "friendly",
        "limit": 3
    })
    
    message = response.json()["message"]
    print(f"宠物说：{message}")
```

#### 3. 用户对话 → 检索记忆

```python
# 用户提问
user_query = "我今天做了什么？"

# 检索相关记忆
response = requests.post(f"{BASE_URL}/retrieve", json={
    "scenario": "conversation",
    "query": user_query,
    "limit": 3
})

memories = response.json()["result"]["items"]

# 构建上下文并调用 LLM
# ...
```

#### 4. 批量处理

```python
# 收集多个观察记录
observations = []

# 每 30 秒收集一次
observations.append({
    "type": "app_switch",
    "app": "VSCode",
    "duration": 300,
    "timestamp": datetime.now().isoformat()
})

# 批量发送
if len(observations) >= 10:
    response = requests.post(f"{BASE_URL}/batch/observations", json={
        "observations": observations
    })
    observations.clear()

# 应用关闭时强制刷新
response = requests.post(f"{BASE_URL}/batch/flush")
```

## 性能建议

1. **使用批量接口**：对于高频监控数据，使用 `/batch/observations` 而不是频繁调用 `/memorize`
2. **检查冷却状态**：调用 `/proactive/analyze` 前先检查 `/proactive/cooldown`，避免生成无效建议
3. **合理设置 limit**：检索时根据实际需求设置 `limit`，避免返回过多数据
4. **应用关闭时刷新**：确保在应用关闭时调用 `/batch/flush`，避免数据丢失

---

## 安全建议

1. **本地部署**：memU-server 应该部署在本地，不要暴露到公网
2. **API Key 保护**：不要在客户端代码中硬编码 API Key
3. **数据加密**：敏感数据应该加密存储
4. **访问控制**：如果需要远程访问，添加身份验证机制
