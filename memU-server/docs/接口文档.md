## 记忆管理接口

### POST /memorize

存储记忆（支持对话和系统观察两种类型）

#### 对话记忆

**请求：**
```json
{
    "messages": [
        {
            "role": "user",
            "content": "我今天工作了 8 小时"
        },
        {
            "role": "assistant",
            "content": "辛苦了，记得休息"
        }
    ]
}
```

**字段说明：**
- `messages`：对话消息数组
  - `role`：角色，可选值：`user`、`assistant`、`system`
  - `content`：消息内容（字符串）

#### 系统观察记录

**请求：**
```json
{
    "type": "system_observation",
    "observation": {
        "type": "app_switch",
        "app": "VSCode",
        "duration": 3600,
        "timestamp": "2024-02-15T10:00:00"
    }
}
```

**字段说明：**
- `type`：固定为 `"system_observation"`
- `observation`：观察记录对象
  - `type`：观察类型（见下表）
  - `timestamp`：时间戳（ISO 8601 格式）
  - 其他字段根据观察类型而定

**支持的观察类型：**

| 类型 | 说明 | 必需字段 | 格式化结果示例 |
|------|------|----------|----------------|
| `app_switch` | 应用切换 | `app`, `duration` | "用户在 VSCode 工作了 60 分钟" |
| `fatigue_detected` | 疲劳检测 | `hours`, `level` | "检测到用户疲劳：连续工作 2 小时" |
| `idle_detected` | 空闲检测 | `minutes` | "用户空闲了 30 分钟" |
| `work_session_end` | 工作会话结束 | `app`, `duration` | "用户结束了在 VSCode 的工作，持续 60 分钟" |
| `late_night_work` | 深夜工作 | `hour` | "检测到深夜工作：23:00 仍在活动" |
| `break_taken` | 休息记录 | `duration` | "用户休息了 10 分钟" |

**响应：**
```json
{
    "status": "success",
    "result": {
        "conversation_id": "uuid",
        "items_created": 3,
        "categories_updated": 2
    }
}
```

**状态码：**
- 200：成功
- 400：请求参数错误
- 500：服务器内部错误

### POST /retrieve

检索记忆（支持对话和主动推理两种场景）

#### 对话场景

用于用户主动提问时检索相关记忆。

**请求：**
```json
{
    "scenario": "conversation",
    "query": "用户今天的工作情况",
    "limit": 3
}
```

**字段说明：**
- `scenario`：固定为 `"conversation"`
- `query`：查询文本
- `limit`：返回记忆数量限制（可选，默认 5）

#### 主动推理场景

用于主动推理时根据上下文检索相关记忆。

**请求：**
```json
{
    "scenario": "proactive",
    "context": {
        "working_duration": 7200,
        "active_app": "VSCode",
        "fatigue_level": "Tired",
        "is_late_night": false,
        "idle_time": 0,
        "is_work_hours": true,
        "focus_level": "NormalFocus"
    },
    "limit": 5
}
```

**字段说明：**
- `scenario`：固定为 `"proactive"`
- `context`：系统上下文对象（见[数据模型](#systemcontext)）
- `limit`：返回记忆数量限制（可选，默认 5）

**响应：**
```json
{
    "status": "success",
    "result": {
        "items": [
            {
                "id": "uuid",
                "summary": "The user wrote 500 lines of Python code today",
                "category": "工作习惯",
                "score": 0.95,
                "created_at": "2024-02-15T10:00:00"
            }
        ],
        "categories": [],
        "resources": []
    }
}
```

**字段说明：**
- `items`：记忆项列表
  - `id`：记忆项 ID
  - `summary`：记忆摘要（英文）
  - `category`：所属类别
  - `score`：相关度分数（0-1）
  - `created_at`：创建时间
- `categories`：相关类别列表
- `resources`：相关资源列表

**状态码：**
- 200：成功
- 400：请求参数错误
- 500：服务器内部错误

## 对话接口

### POST /chat

对话接口 - 支持记忆增强的 AI 对话

**请求：**
```json
{
    "message": "你还记得我今天做了什么吗？",
    "personality": "friendly",
    "use_memory": true,
    "history": [],
    "temperature": 0.7,
    "max_tokens": 2000
}
```

**字段说明：**
- `message`：用户消息（必需）
- `personality`：性格类型（可选，默认 `friendly`，见下表）
- `use_memory`：是否检索相关记忆（可选，默认 `true`）
- `history`：对话历史（可选，数组格式）
- `temperature`：温度参数（可选，默认 0.7）
- `max_tokens`：最大 token 数（可选，默认 2000）

**性格类型：**

| 类型 | 说明 | 语气特点 | 示例 |
|------|------|----------|------|
| `friendly` | 友好型 | 温和友善，使用"呀"、"啦"等语气词 | "主人辛苦啦~ 记得休息眼睛哦" |
| `energetic` | 活力型 | 充满活力，大量感叹号和表情 | "冲鸭主人！今天也要元气满满！" |
| `professional` | 专业型 | 专业简洁，基于数据 | "检测到连续工作2小时。建议：休息10分钟" |
| `tsundere` | 傲娇型 | 表面高冷，内心温柔 | "哼，又来找我了？...算了，休息一下吧" |

**响应：**
```json
{
    "response": "今天你写了 500 行 Python 代码，主要实现了用户认证模块，还学习了 FastAPI 框架呢！",
    "memories_used": 3,
    "memory_used": true
}
```

**字段说明：**
- `response`：AI 回复内容
- `memories_used`：使用的记忆数量
- `memory_used`：是否使用了记忆（布尔值）

**状态码：**
- 200：成功
- 400：请求参数错误
- 500：服务器内部错误

## 主动推理接口

### POST /proactive/analyze

分析当前上下文，生成建议列表（返回结构化数据）

**功能说明：**
- 根据系统上下文（工作时长、疲劳级别等）判断需要什么类型的提醒
- 返回结构化的建议列表（JSON 对象），不是自然语言
- 需要配合 `/proactive/generate` 使用才能生成用户可读的消息

**请求：**
```json
{
    "context": {
        "working_duration": 7200,
        "active_app": "VSCode",
        "fatigue_level": "Tired",
        "is_late_night": false,
        "idle_time": 0,
        "is_work_hours": true,
        "focus_level": "NormalFocus"
    }
}
```

**字段说明：**
- `context`：系统上下文对象（见[数据模型](#systemcontext)）

**响应：**
```json
{
    "status": "success",
    "suggestions": [
        {
            "type": "fatigue_reminder",
            "priority": "high",
            "message": "检测到疲劳",
            "reason": "连续工作 2 小时",
            "action": "建议休息 10 分钟"
        },
        {
            "type": "break_suggestion",
            "priority": "medium",
            "message": "定时休息提醒",
            "reason": "已工作 2 小时",
            "action": "建议站起来活动一下"
        }
    ]
}
```

**注意**：返回的 `message` 字段是建议的简短描述，不是给用户看的自然语言消息。

**建议类型：**
- `fatigue_reminder`：疲劳提醒（冷却时间：30 分钟）
- `late_night_reminder`：深夜提醒（冷却时间：1 小时）
- `idle_interaction`：空闲互动（冷却时间：30 分钟）
- `break_suggestion`：休息建议（冷却时间：30 分钟）
- `hydration_reminder`：喝水提醒（冷却时间：1 小时）

**优先级：**
- `high`：高优先级（需要立即处理）
- `medium`：中优先级（建议处理）
- `low`：低优先级（可选处理）

**状态码：**
- 200：成功
- 500：服务器内部错误

### POST /proactive/generate

根据建议和相关记忆，生成自然语言消息（把结构化建议转换成宠物说的话）

**功能说明：**
- 接收来自 `/proactive/analyze` 的结构化建议
- 结合相关记忆和性格设定
- 生成自然、个性化的消息文本（给用户看的）

**与 `/proactive/analyze` 的区别：**
- `/proactive/analyze`：输入上下文 → 输出结构化建议（JSON）
- `/proactive/generate`：输入结构化建议 → 输出自然语言消息（文本）

**请求：**
```json
{
    "suggestion": {
        "type": "fatigue_reminder",
        "priority": "high",
        "message": "检测到疲劳",
        "reason": "连续工作 2 小时",
        "action": "建议休息 10 分钟"
    },
    "context": {
        "working_duration": 7200,
        "fatigue_level": "Tired"
    },
    "personality": "friendly",
    "limit": 3
}
```

**字段说明：**
- `suggestion`：建议对象（来自 `/proactive/analyze`）
- `context`：系统上下文对象
- `personality`：性格类型（见[对话接口](#post-chat)）
- `limit`：检索记忆数量限制（可选，默认 3）

**响应：**
```json
{
    "status": "success",
    "message": "主人已经工作 2 小时了呀~ 眼睛累了吧？要不要休息一下，我陪你聊会儿天？💕",
    "memories": [
        {
            "id": "uuid",
            "content": "用户通常每 2 小时休息一次",
            "category": "工作习惯"
        }
    ],
    "prompt": "..."
}
```

**字段说明：**
- `message`：生成的自然语言消息（这才是给用户看的）
- `memories`：相关记忆列表
- `prompt`：使用的 LLM prompt（调试用）

**状态码：**
- 200：成功
- 500：服务器内部错误

**示例对比：**

```python
# analyze 返回的是结构化数据
{"message": "检测到疲劳"}  # 简短描述

# generate 返回的是自然语言
{"message": "主人已经工作 2 小时了呀~ 要不要休息一下？💕"}  # 宠物说的话
```

快捷主动推理接口 - 一步完成分析和生成

自动分析上下文，选择最高优先级的建议，并生成自然语言消息。

**请求：**
```json
{
    "context": {
        "working_duration": 7200,
        "active_app": "VSCode",
        "fatigue_level": "Tired",

        "idle_time": 0,
```

**字段说明：**
- `context`：系统上下文对象（见[数据模型](#systemcontext)）
- `personality`：性格类型（可选，默认 `friendly`）
- `limit`：检索记忆数量限制（可选，默认 3）

**响应：**
```json
{
    "status": "success",
    "message": "主人已经工作 2 小时了，要不要休息一下眼睛呀？",
    "suggestion": {
        "type": "fatigue_reminder",
```

- 200：成功

**使用场景：**
- 需要快速获取主动推理消息

## 数据模型

### SystemContext

系统上下文对象，用于主动推理场景。

```typescript
interface SystemContext {
    working_duration: number;      // 工作时长（秒）
    active_app: string;            // 当前活动应用
    fatigue_level: string;         // 疲劳级别
    is_late_night: boolean;        // 是否深夜（22:00-6:00）
    idle_time: number;             // 空闲时间（秒）
    is_work_hours?: boolean;       // 是否工作时间（9:00-18:00）
    focus_level?: string;          // 专注级别
}
```

**疲劳级别（fatigue_level）：**
- `Fresh`：精力充沛（0-1 小时）
- `Normal`：正常（1-2 小时）
- `Tired`：疲劳（2-4 小时）
- `Exhausted`：极度疲劳（4 小时以上）

**专注级别（focus_level）：**
- `DeepFocus`：深度专注（勿扰模式/全屏）
- `NormalFocus`：正常专注
- `Distracted`：分心（频繁切换）
- `Idle`：空闲

---

### Suggestion

建议对象，由 `/proactive/analyze` 生成。

```typescript
interface Suggestion {
    type: string;                  // 建议类型
    priority: string;              // 优先级：high, medium, low
    message: string;               // 建议消息
    reason: string;                // 原因
    action: string;                // 建议的操作
}
```

---

### SystemObservation

系统观察记录对象。

```typescript
interface SystemObservation {
    type: string;                  // 观察类型
    timestamp: string;             // 时间戳（ISO 8601）
    [key: string]: any;            // 其他字段根据类型而定
}
```

**示例：**

```typescript
// 应用切换
{
    type: "app_switch",
    app: "VSCode",
    duration: 3600,
    timestamp: "2024-02-15T10:00:00"
}

// 疲劳检测
{
    type: "fatigue_detected",
    hours: 2,
    level: "Tired",
    timestamp: "2024-02-15T12:00:00"
}
```

## 错误处理

所有接口在出错时返回统一的错误格式：

```json
{
    "detail": "错误信息"
}
```

**HTTP 状态码：**
- 200：成功
- 400：请求参数错误
- 500：服务器内部错误

**常见错误：**

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| "不支持的记忆类型: xxx" | `type` 字段值不正确 | 使用 `conversation` 或 `system_observation` |
| "不支持的检索场景: xxx" | `scenario` 字段值不正确 | 使用 `conversation` 或 `proactive` |
| "对话场景需要提供 'query' 参数" | 缺少 `query` 字段 | 添加 `query` 字段 |
| "message 参数不能为空" | 缺少 `message` 字段 | 添加 `message` 字段 |

## 使用示例

### 完整工作流程

#### 1. 系统监控 → 存储观察

```python
import requests
from datetime import datetime

BASE_URL = "http://localhost:8000"

# 检测到应用切换
observation = {
    "type": "app_switch",
    "app": "VSCode",
    "duration": 3600,
    "timestamp": datetime.now().isoformat()
}

# 存储观察记录
response = requests.post(f"{BASE_URL}/memorize", json={
    "type": "system_observation",
    "observation": observation
})

print(response.json())
```

#### 2. 存储对话记忆

```python
# 存储用户对话
response = requests.post(f"{BASE_URL}/memorize", json={
    "messages": [
        {
            "role": "user",
            "content": "我今天写了 500 行 Python 代码"
        },
        {
            "role": "assistant",
            "content": "很棒！你用的是什么认证方案？"
        }
    ]
})

print(response.json())
```

#### 3. 主动推理 → 生成建议（快捷方式）

```python
# 收集系统上下文
context = {
    "working_duration": 7200,
    "active_app": "VSCode",
    "fatigue_level": "Tired",
    "is_late_night": False,
    "idle_time": 0
}

# 一步完成：分析 + 生成消息
response = requests.post(f"{BASE_URL}/proactive/quick", json={
    "context": context,
    "personality": "friendly",
    "limit": 3
})

result = response.json()
if result["message"]:
    print(f"宠物说：{result['message']}")
    print(f"建议类型：{result['suggestion']['type']}")
else:
    print("当前状态正常，无需主动推理")
```

#### 4. 主动推理 → 生成建议（分步方式）

```python
# 第一步：分析上下文
response = requests.post(f"{BASE_URL}/proactive/analyze", json={
    "context": context
})

suggestions = response.json()["suggestions"]

if suggestions:
    # 选择优先级最高的建议
    suggestion = suggestions[0]
    
    # 第二步：生成自然语言消息
    response = requests.post(f"{BASE_URL}/proactive/generate", json={
        "suggestion": suggestion,
        "context": context,
        "personality": "friendly",
        "limit": 3
    })
    
    message = response.json()["message"]
    print(f"宠物说：{message}")
```

#### 5. 用户对话 → 检索记忆 → AI 回复

```python
# 使用 /chat 接口（推荐）
response = requests.post(f"{BASE_URL}/chat", json={
    "message": "我今天做了什么？",
    "personality": "friendly",
    "use_memory": True
})

result = response.json()
print(f"AI 回复：{result['response']}")
print(f"使用了 {result['memories_used']} 条记忆")
```

#### 6. 手动检索记忆

```python
# 检索相关记忆
response = requests.post(f"{BASE_URL}/retrieve", json={
    "scenario": "conversation",
    "query": "我今天做了什么？",
    "limit": 3
})

memories = response.json()["result"]["items"]
print(f"检索到 {len(memories)} 条相关记忆")

for memory in memories:
    print(f"- [{memory['score']:.2f}] {memory['summary']}")
```

## 接口对比

### 主动推理：快捷 vs 分步

| 方式 | 接口 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| 快捷 | `/proactive/quick` | 一步完成，代码简单 | 无法查看所有建议 | 大多数场景 |
| 分步 | `/proactive/analyze` + `/proactive/generate` | 可查看所有建议，灵活选择 | 需要两次调用 | 需要展示多个建议 |

**推荐**：优先使用 `/proactive/quick`，除非需要展示多个建议供用户选择。

### 主动推理接口详细对比

```
用户场景：工作了 2 小时，需要提醒休息

┌─────────────────────────────────────────────────────────────┐
│ 方式 1：快捷接口（推荐）                                      │
└─────────────────────────────────────────────────────────────┘

POST /proactive/quick
输入：{"context": {"working_duration": 7200}, "personality": "friendly"}
     ↓
输出：{"message": "主人已经工作 2 小时了呀~ 要不要休息一下？💕"}
     ↓
显示给用户


┌─────────────────────────────────────────────────────────────┐
│ 方式 2：分步接口（灵活）                                      │
└─────────────────────────────────────────────────────────────┘

第一步：POST /proactive/analyze
输入：{"context": {"working_duration": 7200}}
     ↓
输出：{
    "suggestions": [
        {"type": "fatigue_reminder", "message": "检测到疲劳"},  ← 结构化数据
        {"type": "break_suggestion", "message": "定时休息"}
    ]
}
     ↓
选择建议：suggestions[0]

第二步：POST /proactive/generate
输入：{"suggestion": suggestions[0], "personality": "friendly"}
     ↓
输出：{"message": "主人已经工作 2 小时了呀~ 要不要休息一下？💕"}  ← 自然语言
     ↓
显示给用户
```

**关键区别：**
- `/proactive/analyze` 返回的是**结构化数据**（JSON 对象），用于程序判断
- `/proactive/generate` 返回的是**自然语言**（文本），用于显示给用户
- `/proactive/quick` 一步完成两个步骤，直接返回自然语言

---

## 重要说明

### 记忆摘要语言

memU 存储的记忆摘要（`summary`）是**英文**，这是 memU 核心库的设计。

- 存储时：中文输入会被提取为英文摘要
- 检索时：中文和英文查询都支持，但中文查询效果更好
- 显示时：建议在前端进行翻译或直接显示英文

**示例：**
```python
# 输入（中文）
"我今天写了 500 行 Python 代码"

# 存储的摘要（英文）
"The user wrote 500 lines of Python code today"

# 检索（中文查询效果更好）
query = "代码"  # ✅ 推荐
query = "Python"  # ✅ 也可以
```

## 性能建议

1. **使用快捷接口**：优先使用 `/proactive/quick` 而不是分步调用
2. **检查冷却状态**：调用主动推理前可以先检查 `/proactive/cooldown`，避免生成无效建议
3. **合理设置 limit**：检索时根据实际需求设置 `limit`，避免返回过多数据
4. **批量存储**：如果有多个观察记录，可以连续调用 `/memorize`，服务端会自动优化

---

## 安全建议

1. **本地部署**：memU-server 应该部署在本地，不要暴露到公网
2. **API Key 保护**：不要在客户端代码中硬编码 API Key
3. **数据加密**：敏感数据应该加密存储
4. **访问控制**：如果需要远程访问，添加身份验证机制
