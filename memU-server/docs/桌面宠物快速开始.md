## 3. 新增功能说明

### 3.1 自定义记忆分类

桌面宠物使用专门的记忆分类：

- 工作习惯
- 情感状态
- 互动历史
- 日常行为
- 兴趣爱好

### 3.2 系统观察记录

支持的观察类型：

| 类型 | 说明 | 示例 |
|------|------|------|
| app_switch | 应用切换 | 用户在 VSCode 工作了 60 分钟 |
| fatigue_detected | 疲劳检测 | 检测到用户疲劳：连续工作 2 小时 |
| idle_detected | 空闲检测 | 用户空闲了 30 分钟 |
| work_session_end | 工作会话结束 | 用户结束了在 VSCode 的工作 |
| late_night_work | 深夜工作 | 检测到深夜工作：23:00 仍在活动 |
| break_taken | 休息记录 | 用户休息了 10 分钟 |

### 3.3 智能检索

#### 对话场景

```python
# 用户询问："我今天做了什么？"
payload = {
    "scenario": "conversation",
    "query": "用户今天的工作情况",
    "limit": 3
}
```

#### 主动推理场景

```python
# 系统检测到用户疲劳，需要检索相关记忆
payload = {
    "scenario": "proactive",
    "context": {
        "working_duration": 7200,  # 2小时
        "fatigue_level": "Tired"
    },
    "limit": 5
}
```

## 4. API 变化

### 4.1 /memorize 接口

新增 `type` 字段：

```python
# 对话记忆（原有功能）
{
    "type": "conversation",
    "content": [...]
}

# 系统观察（新增功能）
{
    "type": "system_observation",
    "observation": {
        "type": "app_switch",
        "app": "VSCode",
        "duration": 3600
    }
}
```

### 4.2 /retrieve 接口

新增 `scenario` 字段：

```python
# 对话场景（原有功能增强）
{
    "scenario": "conversation",
    "query": "查询文本",
    "limit": 3
}

# 主动推理场景（新增功能）
{
    "scenario": "proactive",
    "context": {
        "working_duration": 7200,
        "fatigue_level": "Tired"
    },
    "limit": 5
}
```

## 5. 数据优化

### 5.1 批量存储

- 缓冲区大小：10 条记录
- 刷新间隔：5 分钟
- 自动聚合相似记录

### 5.2 过滤策略

- 应用切换：只保留 >= 5 分钟的记录
- 空闲检测：只保留 >= 30 分钟的记录
- 重要事件：始终保留（疲劳、深夜工作等）

## 7. 常见问题

### Q: 为什么需要适配层？

A: 桌面宠物有特殊需求：
- 高频的系统监控数据
- 主动推理场景的检索
- 自定义的记忆分类

### Q: 适配层会影响性能吗？

A: 不会，反而会提升性能：
- 批量存储减少网络请求
- 数据聚合减少存储量
- 智能检索提高准确性

### Q: 可以直接使用 memU 核心库吗？

A: 可以，但不推荐：
- Electron 主进程是 Node.js，不能直接调用 Python
- memU-server 提供了 HTTP 接口
- 适配层提供了桌面宠物专用的优化
