## 3. 核心功能说明

### 3.1 对话记忆存储

存储用户与桌面宠物的对话：

```cmd
curl -X POST http://127.0.0.1:8000/memorize ^
  -H "Content-Type: application/json" ^
  -d "{\"messages\":[{\"role\":\"user\",\"content\":\"我今天工作了 8 小时\"},{\"role\":\"assistant\",\"content\":\"辛苦了！记得休息哦~\"}]}"
```

### 3.2 系统观察记录

支持的观察类型：

| 类型 | 说明 | 示例 |
|------|------|------|
| app_switch | 应用切换 | 用户在 VSCode 工作了 60 分钟 |
| fatigue_detected | 疲劳检测 | 检测到用户疲劳：连续工作 2 小时 |
| idle_detected | 空闲检测 | 用户空闲了 30 分钟 |
| work_session_end | 工作会话结束 | 用户结束了在 VSCode 的工作 |
| late_night_work | 深夜工作 | 检测到深夜工作：23:00 仍在活动 |
| break_taken | 休息记录 | 用户休息了 10 分钟 |

示例：

```cmd
curl -X POST http://127.0.0.1:8000/memorize ^
  -H "Content-Type: application/json" ^
  -d "{\"type\":\"system_observation\",\"observation\":{\"type\":\"app_switch\",\"app\":\"VSCode\",\"duration\":3600,\"timestamp\":\"2024-02-15T10:00:00\"}}"
```

### 3.3 主动推理

系统可以根据用户状态主动生成关心消息：

```cmd
# 分析当前状态，生成建议
curl -X POST http://127.0.0.1:8000/proactive/analyze ^
  -H "Content-Type: application/json" ^
  -d "{\"context\":{\"working_duration\":7200,\"fatigue_level\":\"Tired\"},\"skip_cooldown\":true}"

# 快捷主动推理（一步完成分析+生成）
curl -X POST http://127.0.0.1:8000/proactive/quick ^
  -H "Content-Type: application/json" ^
  -d "{\"context\":{\"working_duration\":7200,\"fatigue_level\":\"Tired\"},\"personality\":\"friendly\"}"
```

### 3.4 记忆增强对话

对话时自动检索相关记忆：

```cmd
curl -X POST http://127.0.0.1:8000/chat ^
  -H "Content-Type: application/json" ^
  -d "{\"message\":\"你还记得我今天做了什么吗？\",\"personality\":\"friendly\",\"retrieve_memories\":true}"
```

## 4. 核心 API 接口

### 4.1 POST /memorize - 存储记忆

**对话记忆**：
```json
{
    "messages": [
        {"role": "user", "content": "我今天工作了 8 小时"},
        {"role": "assistant", "content": "辛苦了！记得休息哦~"}
    ]
}
```

**系统观察**：
```json
{
    "type": "system_observation",
    "observation": {
        "type": "app_switch",
        "app": "VSCode",
        "duration": 3600,
        "timestamp": "2024-02-15T10:00:00"
    }
}
```

### 4.2 POST /retrieve - 检索记忆

**对话场景**：
```json
{
    "scenario": "conversation",
    "query": "我今天做了什么？",
    "limit": 3
}
```

**主动推理场景**：
```json
{
    "scenario": "proactive",
    "context": {
        "working_duration": 7200,
        "fatigue_level": "Tired"
    },
    "limit": 5
}
```

### 4.3 POST /chat - 记忆增强对话

```json
{
    "message": "你还记得我今天做了什么吗？",
    "personality": "friendly",
    "retrieve_memories": true
}
```

### 4.4 POST /proactive/analyze - 主动推理分析

```json
{
    "context": {
        "working_duration": 7200,
        "fatigue_level": "Tired",
        "is_late_night": false
    },
    "skip_cooldown": true
}
```

返回建议列表（结构化数据）。

### 4.5 POST /proactive/generate - 生成主动推理消息

```json
{
    "suggestion": {
        "type": "fatigue_reminder",
        "priority": "high",
        "message": "检测到疲劳"
    },
    "context": {
        "working_duration": 7200
    },
    "personality": "friendly"
}
```

返回自然语言消息。

### 4.6 POST /proactive/quick - 快捷主动推理

一步完成分析和生成：

```json
{
    "context": {
        "working_duration": 7200,
        "fatigue_level": "Tired"
    },
    "personality": "friendly"
}
```

### 4.7 GET /proactive/cooldown - 获取冷却状态

查询各类建议的冷却状态，防止频繁触发。

## 5. Electron 集成指南

### 5.1 在主进程中创建服务

```typescript
// src/main/services/MemUService.ts
class MemUService {
  private baseURL = 'http://127.0.0.1:8000';
  
  async memorize(messages: Message[]) {
    return await fetch(`${this.baseURL}/memorize`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ messages })
    });
  }
  
  async chat(message: string, personality: string = 'friendly') {
    return await fetch(`${this.baseURL}/chat`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ 
        message, 
        personality,
        retrieve_memories: true 
      })
    });
  }
  
  async proactiveQuick(context: any, personality: string = 'friendly') {
    return await fetch(`${this.baseURL}/proactive/quick`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ context, personality })
    });
  }
}
```

### 5.2 系统监控集成

```typescript
// src/main/services/SystemMonitor.ts
class SystemMonitor {
  private memuService: MemUService;
  
  async recordAppSwitch(app: string, duration: number) {
    await this.memuService.memorize({
      type: 'system_observation',
      observation: {
        type: 'app_switch',
        app,
        duration,
        timestamp: new Date().toISOString()
      }
    });
  }
  
  async checkFatigue(workingDuration: number) {
    const context = {
      working_duration: workingDuration,
      fatigue_level: this.calculateFatigueLevel(workingDuration)
    };
    
    const result = await this.memuService.proactiveQuick(context);
    if (result.message) {
      // 显示主动推理消息
      this.showProactiveMessage(result.message);
    }
  }
}
```

## 7. 常见问题

### Q1: 为什么使用 HTTP API 而不是直接调用 Python？

A: Electron 主进程是 Node.js 环境，无法直接调用 Python 代码。通过 HTTP API：
- 跨语言调用简单
- 服务可以独立部署和扩展
- 便于调试和监控

### Q2: memU-server 需要一直运行吗？

A: 是的，桌面宠物运行时需要 memU-server 提供记忆服务。可以：
- 手动启动：`uv run fastapi dev app/main.py`
- 或在 Electron 启动时自动启动 memU-server 进程

### Q3: 主动推理的冷却机制是什么？

A: 为了避免频繁打扰用户，每种建议类型都有冷却时间：
- 疲劳提醒：30 分钟
- 深夜提醒：1 小时
- 休息建议：30 分钟
- 喝水提醒：1 小时

测试时可以使用 `skip_cooldown: true` 跳过冷却检查。

### Q4: 如何处理 memU-server 连接失败？

A: 建议实现降级策略：
- 检测 memU-server 是否运行（调用 `/health` 接口）
- 如果失败，禁用记忆功能，使用本地对话
- 定期重试连接

### Q5: 记忆检索为什么需要等待？

A: 首次存储记忆后，需要等待 30 秒左右进行向量化处理。后续检索会很快。

### Q6: 如何优化性能？

A: 
- 使用本地 PostgreSQL 数据库（比远程快 5-10 倍）
- 批量存储系统观察（减少 API 调用）
- 合理设置检索 limit（3-5 条即可）
- 使用 `/proactive/quick` 而不是分步调用
