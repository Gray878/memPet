# memU-server 测试指南

## 前置条件

1. 已安装 Python 3.13+
2. 已安装 uv 包管理器
3. 已配置 `.env` 文件
4. PostgreSQL 数据库正在运行

## 步骤 1：检查配置

确保 `.env` 文件配置正确：

```powershell
cd memU-server
type .env
```

应该看到：

```
# 通义千问 API 配置
OPENAI_API_KEY=sk-xxx
OPENAI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
DEFAULT_LLM_MODEL=qwen-plus

# 数据库配置
DATABASE_URL=postgresql+psycopg://postgres:postgres@localhost:5432/memu
```

## 步骤 2：启动 PostgreSQL 数据库

如果使用 Docker：

```powershell
# 进入 memU-server 目录
cd memU-server

# 启动数据库
docker compose up -d postgres

# 检查数据库状态
docker compose ps
```

应该看到 postgres 容器正在运行。

## 步骤 3：启动 memU-server

在第一个终端窗口：

```powershell
# 进入 memU-server 目录
cd memU-server

# 启动服务（开发模式）
uv run fastapi dev app/main.py
```

成功启动后会看到：


## 常见问题

### Q1: 启动时报错 "OPENAI_API_KEY 未设置"

检查 `.env` 文件是否存在，并且包含正确的 API Key。

### Q2: 启动时报错 "数据库配置不完整"

检查 `.env` 文件中的数据库配置，确保 PostgreSQL 正在运行。

### Q3: 测试脚本报错 "无法连接到 http://127.0.0.1:8000"

确保 memU-server 正在运行。检查第一个终端窗口是否有错误信息。

### Q4: 测试返回 500 错误

查看 memU-server 终端的错误日志，通常是：
- API Key 无效
- 数据库连接失败
- LLM 调用失败

### Q5: 记忆存储成功但检索不到（重要）

这是最常见的问题，原因和解决方案：

**原因 1：向量化需要时间**
- memU 需要时间处理记忆并生成向量嵌入（embedding）
- 解决方案：等待 30 秒以上再进行检索测试

**原因 2：数据库中没有数据**
- 记忆可能没有真正存储到数据库
- 解决方案：运行数据库诊断脚本
  ```powershell
  uv run python tests/test_database_check.py
  ```

**原因 3：查询相似度不够**
- 查询文本和记忆内容的相似度太低
- 解决方案：使用更相关的查询文本，或检查嵌入模型配置

**原因 4：memU 判断不需要检索**
- 响应中 `needs_retrieval: False` 表示 memU 认为不需要检索
- 这可能是因为记忆还在处理中，或者查询不够明确

**调试步骤：**
1. 运行数据库诊断：`uv run python tests/test_database_check.py`
2. 检查是否有记忆项存储
3. 增加等待时间到 30 秒
4. 使用更具体的查询文本

### Q6: 测试超时 "Request timed out"

系统观察记录的处理需要调用 LLM 进行分析，可能需要较长时间（30-60秒）。测试脚本已设置 120 秒超时。如果仍然超时，检查：
- 网络连接是否正常
- 通义千问 API 是否可用
- API Key 是否有效

## 查看数据库

如果想查看存储的数据：

```powershell
# 连接到 PostgreSQL
docker exec -it memU-server-postgres-1 psql -U postgres -d memu

# 查看记忆表
\dt

# 查看记忆数据
SELECT * FROM memory_items LIMIT 10;

# 退出
\q
```

## 查看日志

memU-server 的日志会直接输出到终端。如果需要保存日志：

```powershell
uv run fastapi dev app/main.py > server.log 2>&1
```

## 停止服务

### 停止 memU-server

在运行 memU-server 的终端按 `Ctrl+C`

### 停止数据库

```powershell
docker compose down
```
